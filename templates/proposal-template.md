# システム概要書テンプレート

このテンプレートはPhase 1で生成する概要書の構成を定義する。
実際の出力時は以下の3ファイルに分割される：
- `概要_${Project}_${Date}.md`（背景 + 目的）
- `要件_${Project}_${Date}.md`（機能要件 + 非機能要件）
- `業務フロー_${Project}_${Date}.md`（業務フロー）

---

## ファイル3: 業務フロー_${Project}_${Date}.md

```markdown
# ${ProjectName} 業務フロー

作成日: ${Date}
関連文書: 概要_${Project}_${Date}.md, 要件_${Project}_${Date}.md

## 1. 業務フロー概要

### 1.1 全体像
<!-- 業務の全体的な流れを説明 -->
${全体像の説明}

### 1.2 業務フロー図（シーケンス図）

\`\`\`mermaid
sequenceDiagram
    autonumber
    participant ${アクター1} as ${アクター1名}
    participant ${アクター2} as ${アクター2名}
    participant ${システム} as kintone

    ${アクター1}->>+${システム}: ${操作1}（${アプリ名}）
    ${システム}-->>-${アクター1}: ${結果1}

    ${アクター1}->>+${アクター2}: ${依頼内容}

    alt ${条件1}
        ${アクター2}->>+${システム}: ${操作2}
        ${システム}-->>-${アクター2}: ${結果2}
    else ${条件2}
        ${アクター2}->>+${アクター1}: ${差戻し内容}
    end

    Note over ${アクター1},${システム}: ${補足説明}
\`\`\`

## 2. 業務詳細

### 2.1 ${業務名1}

| 項目 | 内容 |
|------|------|
| 担当者 | ${担当者} |
| トリガー | ${開始条件} |
| インプット | ${入力情報} |
| アウトプット | ${出力情報} |
| 関連アプリ | ${アプリ名} |

**処理手順**:
1. ${手順1}
2. ${手順2}
3. ${手順3}

**ステータス遷移**:
${開始ステータス} → ${中間ステータス} → ${完了ステータス}

## 3. ステータス定義

| ステータス名 | 説明 | 次のステータス | 遷移条件 |
|--------------|------|----------------|----------|
| ${ステータス1} | ${説明1} | ${次1} | ${条件1} |
| ${ステータス2} | ${説明2} | ${次2} | ${条件2} |

## 4. 承認フロー（該当する場合）

### 4.1 承認フロー図

\`\`\`mermaid
sequenceDiagram
    autonumber
    participant 申請者 as ${申請者}
    participant 承認者1 as ${承認者1}
    participant 承認者2 as ${承認者2}
    participant kintone as kintone

    申請者->>+kintone: 申請登録
    kintone-->>-申請者: 登録完了（ステータス: 申請中）

    kintone->>承認者1: 承認依頼通知

    alt 承認
        承認者1->>+kintone: 承認処理
        kintone-->>-承認者1: 承認完了（ステータス: 一次承認済）
        kintone->>承認者2: 承認依頼通知

        alt 最終承認
            承認者2->>+kintone: 最終承認
            kintone-->>-承認者2: 完了（ステータス: 承認済）
            kintone->>申請者: 承認完了通知
        else 却下
            承認者2->>+kintone: 却下処理
            kintone-->>-承認者2: 却下完了（ステータス: 却下）
            kintone->>申請者: 却下通知
        end
    else 差戻し
        承認者1->>+kintone: 差戻し処理
        kintone-->>-承認者1: 差戻し完了（ステータス: 差戻し）
        kintone->>申請者: 差戻し通知
        Note over 申請者,kintone: ${差戻し時の対応}
    end
\`\`\`

### 4.2 承認ルート一覧

| 申請者 | 承認者1 | 承認者2 | 条件 |
|--------|---------|---------|------|
| ${申請者1} | ${承認者1} | ${承認者2} | ${条件} |
```

## 記述ルール

1. **技術用語**: 括弧書きで補足説明を入れる
   - 例: ルックアップ（他のアプリから情報を自動取得する機能）

2. **曖昧な表現の回避**: 具体的な数値や条件を記載
   - NG: 「多くのデータ」
   - OK: 「月間約1,000件のデータ」

3. **変数の命名**: `${変数名}` 形式で記載し、生成時に置換
