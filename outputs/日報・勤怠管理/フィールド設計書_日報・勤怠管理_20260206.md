### 4.4 計算フィールド

#### 勤務時間（calc_work_hours）

| 項目 | 設定値 |
|------|--------|
| 計算式 | `IF(退勤時刻 = "", 0, (HOUR(退勤時刻) * 60 + MINUTE(退勤時刻) - HOUR(出勤時刻) * 60 - MINUTE(出勤時刻) - 休憩時間) / 60)` |
| 小数点以下 | 2桁 |
| 単位 | 時間 |

**計算ロジック説明**:
- 退勤時刻が未入力の場合は0を返す
- 退勤時刻と出勤時刻を分に変換して差分を計算
- 休憩時間（分）を差し引く
- 60で割って時間単位に変換

### 5.3 カスタマイズ詳細

#### C-001: 日報承認後編集不可

| 項目 | 設定値 |
|------|--------|
| パターンID | field_disable |
| 対象アプリ | 日報管理（daily_report） |
| トリガーイベント | app.record.edit.show |
| トリガー条件 | ステータス = "承認済" |
| 対象フィールド | report_date, employee_id, project_code, work_content, work_hours, progress, tomorrow_plan, notes |
| 処理内容 | 対象フィールドを編集不可にする |

**実装メモ**:
- プロセス管理のステータスが「承認済」の場合、入力フィールドを全て編集不可にする
- 編集画面表示時（app.record.edit.show）にフィールドをdisableにする
- システムフィールド、ルックアップコピー先フィールドは対象外（元々編集不可のため）

#### C-002: 勤怠承認後編集不可

| 項目 | 設定値 |
|------|--------|
| パターンID | field_disable |
| 対象アプリ | 勤怠管理（attendance） |
| トリガーイベント | app.record.edit.show |
| トリガー条件 | ステータス = "承認済" |
| 対象フィールド | attendance_date, employee_id, clock_in, clock_out, break_time, work_type, notes |
| 処理内容 | 対象フィールドを編集不可にする |

**実装メモ**:
- プロセス管理のステータスが「承認済」の場合、入力フィールドを全て編集不可にする
- 編集画面表示時（app.record.edit.show）にフィールドをdisableにする
- 計算フィールド（calc_work_hours）、システムフィールド、ルックアップコピー先フィールドは対象外

#### C-003: 退勤時刻の条件付き必須

| 項目 | 設定値 |
|------|--------|
| パターンID | validate_required |
| 対象アプリ | 勤怠管理（attendance） |
| トリガーイベント | app.record.edit.submit, app.record.create.submit |
| トリガー条件 | 勤務種別 in ("通常勤務", "在宅勤務", "時短勤務") |
| 対象フィールド | clock_out |
| 処理内容 | 退勤時刻が未入力の場合、エラーメッセージを表示して保存を中止 |

**実装メモ**:
- 有給休暇、特別休暇、欠勤の場合は退勤時刻は不要
- 通常勤務、在宅勤務、時短勤務の場合は退勤時刻を必須とする

## 6. フィールドコード命名規則

本設計書で使用したフィールドコードは以下の命名規則に従っています。

1. **基本ルール**:
   - 英数字とアンダースコアのみ使用
   - 小文字で統一
   - 単語間はアンダースコアで区切る

2. **命名パターン**:
   - 基本項目: `{項目名}` 例: `employee_name`, `report_date`
   - IDフィールド: `{対象}_id` 例: `employee_id`, `project_code`
   - 計算フィールド: `calc_{項目名}` 例: `calc_work_hours`
   - 時刻フィールド: `clock_{in/out}` 例: `clock_in`, `clock_out`

1. **ルックアップフィールドの絞り込み**:
   - 社員マスタのルックアップでは「在籍」状態の社員のみ選択可能
   - 案件マスタのルックアップでは「見積中」「進行中」の案件のみ選択可能

2. **プロセス管理の作業者**:
   - 上長フィールド（manager）をプロセス管理の承認者として使用
   - ルックアップで社員マスタから自動取得されるため、承認者の手動設定は不要

3. **勤務時間の計算**:
   - 日をまたぐ勤務（深夜勤務）には非対応
   - 必要に応じてカスタマイズで対応可能

4. **カスタマイズの適用順序**:
   - Phase 3でアプリデプロイ後、Phase 3.5でカスタマイズコードを適用
   - 新規アプリのみカスタマイズを適用（既存カスタマイズがある場合はスキップ）
