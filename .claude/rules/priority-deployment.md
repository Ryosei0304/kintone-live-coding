# 2-Pass デプロイルール

## なぜ2パスが必要か

kintoneでは参照先アプリがデプロイ済みでないとルックアップ/関連レコードを設定できない。
このプロジェクトでは **2-Pass デプロイ** でこの問題を解決する。

## Pass 1: 基本フィールド

1. 全アプリを作成（プレビュー状態）
2. 基本フィールド（ルックアップ/関連レコード以外）を追加
3. 全アプリをデプロイ

## Pass 2: 連携フィールド（優先度順）

1. ルックアップフィールドを優先度順に追加
2. 関連レコード一覧フィールドを追加
3. フォームレイアウトを最適化
4. 全アプリを再デプロイ

## デプロイ優先度

以下の優先度に従ってデプロイ順序を決定する。

1. **マスタアプリ優先**: 他のアプリから参照されるマスタアプリを先にデプロイ
2. **参照先 → 参照元の順**: ルックアップ/関連レコードの参照先アプリが先、参照元アプリが後
3. **循環参照はエラー**: 循環参照がある場合はエラーとする

## 進捗表示例

```
[Pass 1] 基本フィールド
  アプリ作成 (1/3): 顧客マスタ
  アプリ作成 (2/3): 商品マスタ
  アプリ作成 (3/3): 受注管理
  デプロイ: 完了

[Pass 2] 連携フィールド（優先度順）
  顧客マスタ → 企業マスタ (ルックアップ): OK
  受注管理 → 顧客マスタ (ルックアップ): OK
  受注管理 → 商品マスタ (ルックアップ): OK
  デプロイ: 完了
```
