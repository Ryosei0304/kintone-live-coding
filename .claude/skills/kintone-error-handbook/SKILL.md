# kintone エラーハンドブック

kintone REST APIのエラーコード別対処法をまとめたリファレンスです。

## 概要

このスキルは、kintone REST API使用時に発生するエラーの：
- 即座の原因特定
- 対処法の迅速な適用
- 同じエラーの再発防止

を支援します。

## エラーコード一覧

### フィールド関連エラー

#### GAIA_FC01: 指定されたフィールドが見つかりません

| 項目 | 内容 |
|------|------|
| **発生タイミング** | ルックアップ/関連レコード一覧の追加時 |
| **原因** | 参照先フィールドがデプロイされていない |
| **対処法** | 参照先アプリをデプロイしてから再実行 |

**詳細**:
- ルックアップの`relatedKeyField`や`fieldMappings`で指定したフィールドが本番環境に存在しない
- 関連レコード一覧の`condition.relatedField`や`displayFields`で指定したフィールドが本番環境に存在しない

**解決手順**:
1. 参照先アプリのデプロイ状態を確認
2. 参照先フィールド（特にルックアップフィールド）がデプロイ済みか確認
3. 未デプロイの場合は先にデプロイ
4. 再度ルックアップ/関連レコード一覧を追加

---

#### GAIA_LO03: ルックアップのキーフィールドに重複禁止設定がない

| 項目 | 内容 |
|------|------|
| **発生タイミング** | ルックアップフィールドの追加時、テストデータ投入時 |
| **原因** | 参照先のキーフィールドに`unique: true`が設定されていない |
| **対処法** | 参照先フィールドに重複禁止設定を追加 |

**詳細**:
- ルックアップの参照先フィールドは「値の重複を禁止する」設定が必須
- フィールド設計書で`重複禁止: ○`になっているか確認

**解決手順**:
1. 参照先アプリのキーフィールドを確認
2. `unique: true`を設定してフィールドを更新
3. アプリをデプロイ
4. 再度ルックアップを追加

---

#### GAIA_IL23: フィールドコードが重複しています

| 項目 | 内容 |
|------|------|
| **発生タイミング** | フィールド追加時 |
| **原因** | 同じフィールドコードが既に存在する |
| **対処法** | 一意なフィールドコードを指定 |

**詳細**:
- アプリ内でフィールドコードは一意である必要がある
- システムフィールド（ステータス、作成者など）と同じコードは使用不可

**解決手順**:
1. 既存フィールドのコードを確認（`kintone-get-form-fields`）
2. 重複しないコードに変更
3. 再度フィールドを追加

### リクエスト形式エラー

#### CB_VA01: 入力値が不正です

| 項目 | 内容 |
|------|------|
| **発生タイミング** | 各種API呼び出し時 |
| **原因** | リクエストパラメータの形式が不正 |
| **対処法** | パラメータを確認して修正 |

**よくあるケース**:

1. **サブテーブル内フィールドの個別更新**
   - 「テーブルからフィールドを出すことはできません」
   - サブテーブル内フィールドは個別に更新できない
   - サブテーブル全体を再定義して更新する

2. **フィールドコードの形式不正**
   - 数字で始まるコード
   - 使用禁止文字（@, +, ~, #など）を含むコード

3. **必須パラメータの欠落**
   - ドロップダウンに`options`がない
   - ルックアップに`lookup`設定がない

**解決手順**:
1. エラーメッセージの詳細を確認
2. リクエストボディの形式を確認
3. kintone APIドキュメントと照合
4. 修正して再実行

---

#### CB_IJ01: 不正なJSON形式です

| 項目 | 内容 |
|------|------|
| **発生タイミング** | 各種API呼び出し時 |
| **原因** | JSONの構文エラー |
| **対処法** | JSONを検証して修正 |

**よくあるケース**:
- カンマの過不足
- 引用符の不一致
- エスケープ不足

**解決手順**:
1. JSONをバリデーターで検証
2. 構文エラーを修正
3. 再実行

---

### アプリ・レコード関連エラー

#### CB_NO02: 指定されたアプリが見つかりません

| 項目 | 内容 |
|------|------|
| **発生タイミング** | アプリIDを指定するAPI呼び出し時 |
| **原因** | アプリIDが存在しない、またはアクセス権がない |
| **対処法** | アプリIDを確認 |

**解決手順**:
1. アプリIDが正しいか確認
2. アプリがデプロイ済みか確認（プレビュー状態では一部APIが使えない）
3. アクセス権を確認

---

#### GAIA_AP01: アプリの操作権限がありません

| 項目 | 内容 |
|------|------|
| **発生タイミング** | アプリ設定変更時 |
| **原因** | ユーザーにアプリ管理権限がない |
| **対処法** | kintone管理者に権限付与を依頼 |

**必要な権限**:
- アプリ作成: cybozu.com共通管理者権限またはスペース管理者権限
- フィールド追加/変更: アプリ管理者権限
- レコード操作: アプリのレコード権限

### 同時アクセス・制限関連エラー

#### 429: 同時アクセス上限超過

| 項目 | 内容 |
|------|------|
| **発生タイミング** | 大量のAPI呼び出し時 |
| **原因** | 同時アクセス数が上限（100）を超えた |
| **対処法** | リトライ処理（指数バックオフ）を実装 |

**制限値**:
- ドメインごとの同時アクセス数: 100
- 1アプリへのリクエスト数: 10,000リクエスト/日

**解決手順**:
1. リクエスト間隔を空ける
2. 指数バックオフでリトライ
3. バッチサイズを小さくする

---

#### 520: アプリ作成上限

| 項目 | 内容 |
|------|------|
| **発生タイミング** | アプリ作成時 |
| **原因** | 契約プランのアプリ数上限に達した |
| **対処法** | 不要アプリの削除、契約プラン確認 |

### デプロイ関連エラー

#### デプロイが完了しない

| 項目 | 内容 |
|------|------|
| **発生タイミング** | `deploy.json`呼び出し後 |
| **原因** | デプロイは非同期処理のため、完了を待つ必要がある |
| **対処法** | ステータス確認APIでポーリング |

**解決手順**:
```bash
# デプロイ実行
curl -s -X POST "${KINTONE_DOMAIN}/k/v1/preview/app/deploy.json" \
  -H "X-Cybozu-Authorization: ${AUTH}" \
  -H "Content-Type: application/json" \
  -d '{"apps": [{"app": "123"}]}'

# ステータス確認（SUCCESSになるまで繰り返し）
curl -s -X GET "${KINTONE_DOMAIN}/k/v1/preview/app/deploy.json?apps[0]=123" \
  -H "X-Cybozu-Authorization: ${AUTH}"
```

## エラー発生時のデバッグ手順

### 1. エラーメッセージの確認

```bash
# レスポンスボディとHTTPステータスを両方取得
RESPONSE=$(curl -s -w "\n%{http_code}" -X POST "${KINTONE_DOMAIN}/k/v1/..." \
  -H "X-Cybozu-Authorization: ${AUTH}" \
  -H "Content-Type: application/json" \
  -d '...')

HTTP_CODE=$(echo "$RESPONSE" | tail -n1)
BODY=$(echo "$RESPONSE" | sed '$d')

echo "HTTP Status: $HTTP_CODE"
echo "Response: $BODY"
```

### 2. エラーコードの特定

レスポンスボディから以下を確認：
- `code`: エラーコード（GAIA_FC01など）
- `message`: エラーメッセージ
- `errors`: 詳細なエラー情報（存在する場合）

### 3. 対処法の適用

このハンドブックのエラーコードセクションを参照し、対処法を適用。
