# Phase 1: コンテキスト把握（WHO / WHY）

ペインポイント起点の最小限ヒアリングで、アクター・課題・ゴール・スコープを整理します。

## 概要

このスキルは `/kintone-workflow` から呼び出され、以下を実行します：
1. 最小限の質問（3問以内）でコンテキストを把握
2. サマリーしてユーザーに確認
3. コンテキスト整理書を生成

出力: `コンテキスト整理_${Project}_${Date}.md`

## 入力

- プロジェクト名
- 出力日付（YYYYMMDD形式）

## ヒアリング方針（重要）

**質問は3問以内。ペインポイント起点で聞く。**

### ヒアリングの流れ

1. **Q1: 業務とペインポイント** → 回答を待つ
2. **Q2: アクター** → 回答を待つ
3. **Q3（任意）: 最優先ゴール** → Q1で明確ならスキップ

詳細は `.claude/agents/reference/context-interview-flow.md` を参照。

### 聞かないこと

以下はPhase 2で業務フロー理解後に聞く方が的確な回答が得られるため、**聞かない**:
- 管理したいデータ項目の詳細
- 利用規模・データ量
- 外部連携・セキュリティ
- カスタマイズ要件

### サマリー確認

全質問完了後、収集した情報をサマリーしてユーザーに確認:

```
ここまでのお話を整理しますね！

【アクター】
- ...

【ペインポイント】
- ...

【ゴール】
- ...（優先度: 高）

【スコープ（仮）】
- 対象: ...
- 対象外: ...

この理解で合っていますか？修正や追加があれば教えてください！
```

## 生成手順

### 1. テンプレート読み込み

`templates/context-template.md` を参照し、構造を理解する。

### 2. 情報の整理・構造化

収集した情報を以下のカテゴリに分類：

**アクター**: 誰が使うか、部署・役職、主な役割
**ペインポイント**: 何に困っているか、影響度、現状の対応方法
**ゴール**: 何を解決したいか、優先度、達成基準
**スコープ（仮）**: 対象範囲、対象外、制約・前提条件

### 3. ファイル生成

Writeツールを使用して1ファイルを生成:

- `${OutputDir}コンテキスト整理_${Project}_${Date}.md`

### 4. 生成結果の表示

生成完了後、以下を表示:
- 生成したファイル
- アクター数、ペインポイント数、ゴール数

## セルフチェックリスト

- [ ] アクターが具体的に記述されている（部署・役職レベル）
- [ ] ペインポイントが影響度付きで記録されている
- [ ] ゴールに優先度が設定されている
- [ ] スコープ（仮）の対象/対象外が明記されている

## 注意事項

1. **聞きすぎない**: 3問以内で終わらせる。詳細はPhase 2で聞く
2. **曖昧な表現の回避**: 具体的な数値・条件を使用
3. **技術用語の補足**: 非エンジニア向けに括弧書きで説明を追加
4. **スコープは仮**: Phase 2で業務フローを整理した後に確定する
