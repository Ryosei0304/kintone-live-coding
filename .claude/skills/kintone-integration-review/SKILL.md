# Phase 4: 統合レビュー

Phase 1-3で作成した全文書を横断的に検証し、ギャップ・矛盾を検出します。

## 概要

このスキルは `/kintone-workflow` のオーケストレーターが直接実行し、以下を検証します：
1. 業務フロー × アプリ × ユースケースのクロスバリデーション
2. ギャップ・矛盾の検出
3. デプロイ計画の確認
4. 統合レビュー結果の生成

出力: `統合レビュー_${Project}_${Date}.md`

## 入力

以下を全て読み込むこと:
- `${OutputDir}コンテキスト整理_${Project}_${Date}.md`
- `${OutputDir}業務フロー設計_${Project}_${Date}.md`
- `${OutputDir}アプリアーキテクチャ_${Project}_${Date}.md`
- `${OutputDir}アプリ設計書_${Project}_${Date}.md`
- `${OutputDir}フィールド設計書_${Project}_${Date}.md`

## クロスバリデーション項目

### 1. ビジネスイベント → アプリ カバレッジ

業務フロー設計のビジネスイベントが全てアプリアーキテクチャでカバーされているか。

| チェック | 方法 |
|---------|------|
| 漏れなし | 全ビジネスイベントがいずれかのアプリに対応 |
| 過不足なし | 不要なアプリがないか（ビジネスイベントに紐付かないアプリ） |

### 2. ユースケース → アプリ カバレッジ

全ユースケースが実行可能な設計になっているか。

| チェック | 方法 |
|---------|------|
| 全ユースケースが実行可能 | 必要なアプリ・フィールドが存在する |
| アクターの操作が成立 | 権限設計でアクターが必要な操作を実行可能 |

### 3. アーキテクチャ → 詳細設計 整合性

アプリアーキテクチャのキーフィールドが詳細設計に反映されているか。

| チェック | 方法 |
|---------|------|
| キーフィールド一致 | アーキテクチャのPK/FKがフィールド設計書に存在 |
| ルックアップ設定一致 | ER図の連携がフィールド設計書のルックアップ設定と一致 |
| ステータス一致 | アーキテクチャのステータスフィールドがプロセス管理設定と一致 |

### 4. カスタマイズ要件 カバレッジ

業務フロー設計のカスタマイズ要件が詳細設計に反映されているか。

| チェック | 方法 |
|---------|------|
| 全要件がマッピング済み | アーキテクチャのマッピングがフィールド設計書に反映 |
| 対象フィールドが存在 | カスタマイズ対象のフィールドコードがフィールド設計書に存在 |
| パターンIDが有効 | 指定されたパターンIDが `templates/customize/catalog.json` に存在 |

### 5. デプロイ順序の整合性

| チェック | 方法 |
|---------|------|
| 循環参照なし | ルックアップの依存関係に循環がない |
| 参照先が先にデプロイ | マスタアプリが先、トランザクションアプリが後 |
| 2-Pass方式に整合 | Pass 1で基本フィールド、Pass 2でルックアップ/関連レコード |

## ギャップ分類

| カテゴリ | 説明 | Phase 5への影響 |
|---------|------|---------------|
| CRITICAL | ブロッカー。修正しないとデプロイ不可 | Phase 5に進めない |
| WARNING | 要検討。デプロイは可能だが品質リスクあり | 確認の上で進行可能 |
| INFO | 参考情報。改善の余地はあるが問題なし | 影響なし |

## 生成手順

### 1. テンプレート読み込み

`templates/integration-review-template.md` を参照。

### 2. クロスバリデーション実行

上記チェック項目を順番に検証。

### 3. 統合レビュー文書生成

- `${OutputDir}統合レビュー_${Project}_${Date}.md`

## 注意事項

1. **CRITICALが0件であること**: CRITICALが残っている場合はPhase 5に進めない
2. **WARNINGはユーザー判断**: ユーザーに提示して進行可否を判断してもらう
3. **既存スキルの活用**: 詳細設計の品質チェックは `kintone-design-review` も併用
