# kintone 既存アプリ修正ワークフロー開始

既存のkintoneアプリを読み込み、変更を適用するワークフローを開始します。

## 実行内容

`/kintone-restart-workflow` スキルを実行してください。

このワークフローでは以下のフェーズを順番に実行します：

1. **Step 0（アプリ選択）**: スペースID指定 → アプリ一覧表示 → 対象アプリ選択
2. **Phase R1（リバースエンジニアリング）**: 既存アプリ読み込み → 現状分析書生成
3. **Phase R2（変更計画作成）**: ユーザーの変更要望 → 構造化された変更計画
4. **Phase R3（詳細設計更新）**: 設計書の生成・更新
5. **Phase R4（変更適用）**: kintoneに変更を適用 + カスタマイズ
